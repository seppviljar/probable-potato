Chave da API - Um identificador exclusivo de uma integração e um parâmetro necessário para autenticação. É usada para criar o valor do cabeçalho X-AUTH-CLIENT para solicitações da API. Você pode encontrá-la no ambiente da Veriff > página de Integração (requer login). Ocasionalmente, referida como a chave pública da API ou chave publicável.

Solicitação da API - Uma mensagem enviada a um servidor pedindo que a API forneça serviço ou informação. Também referida como a chamada da API.

Arquivo -

AttemptId - Um UUID v4 de uma tentativa específica. Você pode encontrar seu valor na resposta do GET /sessions/{sessionId}/attempts, como {verification.id}. No contexto das solicitações da API, uma "tentativa" é um passo (upload dos dados requeridos e recebendo uma decisão) dentro de uma sessão de verificação, e pode ser feita várias vezes durante uma sessão de verificação. Isso significa que chamar o GET /sessions/{sessionId}/attempts pode retornar vários objetos {verification}, cada um com um valor {id} único.

SESSÃO COM CUSTO - Uma sessão de fornecedor que foi realizada na integração ao vivo e tem pelo menos uma decisão final - aprovada, recusada ou re-submissão solicitada. As sessões com custo são relatadas na marcação de Primeiro Verificado Em.

Sessão -

sessionID - Um identificador único de uma sessão de verificação. É criado automaticamente assim que você cria uma sessão. Você pode encontrar seu valor no payload de resposta de sua sessão POST, como verification.id. No contexto das solicitações da API, uma sessão de verificação compreende muitos passos (upload de dados, upload de imagens, recebendo uma decisão, etc.). Você pode chamar vários endpoints usando o mesmo {sessionID}, para obter todos os tipos de dados diferentes daquela sessão específica.

Status da sessão - Um rótulo que descreve o status de uma sessão de verificação.

Chave secreta compartilhada - Parâmetro necessário para autenticação, usado para assinar o payload para criar o X-HMAC-SIGNATURE para as solicitações da API. Ocasionalmente, referida como a chave privada da API ou a Chave Privada.

Pacotes Iniciais

SUBMETIDO - estado submetido: quando o usuário envia os documentos e conclui o fluxo.

SESSÕES DE VERIFICAÇÃO SUBMETIDAS - Sessão de verificação que o usuário final submeteu. Sessão de verificação com status: aprovado, recusado, re-submissão solicitada, pré-processada ou submetida.

Integração de Teste - É usada para fins de desenvolvimento, e as sessões de verificação que são criadas não contarão para uso pago. A Veriff não fornecerá status para sessões criadas na integração de teste, você precisará defini-los manualmente na Station (veja aqui para instruções). A URL de decisão do Webhook é criada automaticamente.

Fornecedor -

FORNECEDOR - Uma entidade comercial QUE VENDE serviços ou produtos para a Veriff (ou seja, não um Cliente ou Cliente)

Veriff Station - O ambiente da Veriff, um painel onde os clientes podem ver os dados de verificação de seus usuários finais. Dependendo da sua configuração, pode ser necessário acessar o ambiente pelo station.veriff.com ou hub.veriff.com. Verifique seu e-mail de inscrição para ter certeza de que você faça login pelo endereço correto.

Fluxo de verificação - Um processo onde o usuário final envia os dados requeridos para ser verificado.

SESSÃO DE VERIFICAÇÃO - corresponde a uma tentativa de verificação única por usuário_final. Uma sessão_de_fornecedor pode ter várias sessões_de_verificação caso o usuário tenha tentativas de verificação malsucedidas que resultaram em decisões inconclusivas.

Sessão de verificação - Uma sessão de verificação do início ao fim, ou seja, desde o momento em que uma sessão é criada até o momento em que uma decisão conclusiva (“aprovado”, “recusado”, “expirada”/“abandonada”) é concedida. Cada sessão de verificação recebe um ID único, também conhecido como {sessionID} usado nos endpoints de solicitação da API.

SESSÕES VERIFICADAS - O usuário final recebe uma decisão de um conjunto de {aprovado, recusado, re-submissão_solicitada, revisão}

Webhook - Uma API acionada por evento, significando que em vez de enviar informações ou executar uma função como resposta a uma solicitação de aplicativo, ela executa isso quando um determinado evento de disparo acontece.

Cabeçalho X-AUTH-CLIENT - Um campo de cabeçalho obrigatório nas solicitações da API, necessário para identificar o remetente da solicitação. Seu valor é a Chave da API (veja acima).

Cabeçalho X-HMAC-SIGNATURE - Um campo de cabeçalho obrigatório nas solicitações da API, necessário para autenticar o remetente da solicitação. Seu valor é um valor hash HMAC-SHA256 codificado em hex dos payloads da solicitação que foram assinados usando uma chave secreta compartilhada conhecida tanto pelo remetente quanto pelo destinatário.
