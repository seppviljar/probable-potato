AttemptId - Un UUID v4 de un intento específico. Puedes encontrar su valor en tu respuesta GET /sessions/{sessionId}/attempts, como {verification.id}. En el contexto de las solicitudes de la API, un "intento" es un paso (subir datos requeridos y obtener una decisión) dentro de una sesión de verificación, y puede realizarse varias veces durante una sesión de verificación. Esto significa que llamar a GET /sessions/{sessionId}/attempts puede devolver varios objetos {verification}, cada uno con un valor {id} único.

Sesión -

sessionID - Un identificador único de una sesión de verificación. Se crea automáticamente tan pronto como creas una sesión. Puedes encontrar su valor en la carga útil de respuesta de tu sesión POST, como verification.id. En el contexto de las solicitudes de la API, una sesión de verificación comprende muchos pasos (subir datos, subir imágenes, obtener una decisión, etc.). Puedes llamar a varios endpoints usando el mismo {sessionID}, para obtener todo tipo de datos diferentes de esa sesión específica.

Estado de la sesión - Una etiqueta que describe el estado de una sesión de verificación.

Clave compartida secreta - Parámetro requerido para la autenticación, utilizado para firmar la carga útil para crear la X-HMAC-SIGNATURE para solicitudes API. Ocasionalmente, se hace referencia a ella como la clave privada de la API o la clave privada.

Paquetes básicos

ENVIADO - estado enviado: cuando el usuario envía los documentos y completa el flujo.

SESIONES DE VERIFICACION ENVIADAS - Sesión de verificación que el usuario final ha enviado. Sesión de verificación con estado: aprobada, rechazada, reenvío solicitado, preprocesada o enviada.

Integración de prueba - Se utiliza con fines de desarrollo, y las sesiones de verificación que se creen no contarán para el uso pagado. Veriff no proporcionará estados para sesiones creadas en la integración de prueba, debes configurarlas manualmente en Station (ver aquí para instrucciones). La URL de decisión del Webhook se crea automáticamente.

Proveedor -

PROVEEDOR - Una entidad comercial que VENDE servicios o bienes a Veriff (es decir, no un Cliente o Cliente)

Veriff Station - El entorno de Veriff, un panel donde los clientes pueden ver los datos de verificación de sus usuarios finales. Dependiendo de tu configuración, es posible que necesites acceder al entorno a través de station.veriff.com o hub.veriff.com. Por favor, revisa tu correo electrónico de inscripción para asegurarte de iniciar sesión a través de la dirección correcta.

Flujo de verificación - Un proceso en el que el usuario final envía los datos requeridos para ser verificado.

SESION DE VERIFICACION - corresponde a un intento de verificación único por el usuario final. Una sesión de proveedor puede tener múltiples sesiones de verificación en caso de que el usuario tenga intentos de verificación no exitosos que resultaron en decisiones inconclusas.

Sesión de verificación - Una sesión de verificación de principio a fin, es decir, desde el momento en que se crea una sesión hasta el momento en que se otorga una decisión concluyente (“aprobada”, “rechazada”, “expirada”/“abandonada”). Cada sesión de verificación recibe un ID único, también conocido como {sessionID} utilizado en los endpoints de solicitud API.

SESIONES VERIFICADAS - El usuario final recibe una decisión de un conjunto de {aprobada, rechazada, reenvío solicitado, revisión}

Webhook - Una API activada por eventos, lo que significa que en lugar de enviar información o realizar una función como respuesta a una solicitud de una aplicación, lo realiza cuando ocurre un evento desencadenante específico.

Encabezado X-AUTH-CLIENT - Un campo de encabezado obligatorio en las solicitudes de API, requerido para identificar al remitente de la solicitud. Su valor es la API Key (ver arriba).

Encabezado X-HMAC-SIGNATURE - Un campo de encabezado obligatorio en las solicitudes de API, necesario para autenticar al remitente de la solicitud. Su valor es un valor hash en hexadecimales de la carga útil de la solicitud que ha sido firmada utilizando una clave compartida secreta conocida por ambas partes, la que envía y la que recibe.
