API Key - A unique identifier of an integration and a required parameter for authentication. It is used to create the X-AUTH-CLIENT header value for API requests. You can find it in the Veriff environment > Integration's page (requires a login). Occasionally, referred to as the API public key or Publishable key.

API request - A message sent to a server asking for the API to provide service or information. Also referred to as the API call.

AttemptId - An UUID v4 of a specific attempt. You can find its value in your GET /sessions/{sessionId}/attempts response, as {verification.id}. In the API requests\' context, an "attempt" is one step (uploading required data and getting a decision) inside a verification session, and it can be made several times during a verification session. This means that calling the GET /sessions/{sessionId}/attempts can return several {verification} objects, each one with a unique {id} value.

BILLABLE SESSION - A vendor session that has been done in live integration and has at least one final decision - approved, declined or resubmission requested. Billable sessions are reported on First Verified At timestamp.

CallbackURL - URL address of the page where the end-user is sent after they've completed the web verification flow.

Content-type header - Indicates the media type of the resource, in Veriff's context, usually "application/json".

Customer - Customer is the legal entity and those of its affiliates to whom Veriff provides the service.

Session -

sessionID - A unique identifier of a verification session. It is automatically created as soon as you create a session. You can find its value in the response payload of your POST session, as verification.id. In the API requests' context, one verification session comprises many steps (uploading data, uploading images, getting a decision, etc.). You can call several endpoints using the same {sessionID}, to get all kinds of different data from that specific session.

Session status - A label describing the status of a verification session.

Shared secret key - Required parameter for authentication, used to sign the payload to create the X-HMAC-SIGNATURE for API requests. Occasionally, referred to as the API private key or the Private key.

SSN - (Social Security number) Verification

STARTED SESSIONS - End user clicks on the flow to start the session

Starter Packages

SUBMITTED - submitted state: when user submits the documents and completes the flow.

SUBMITTED VERIFICATION SESSIONS - Verification session that the end-user has submitted. Verification session with status: approved, declined, resubmission requested, preprocessed, or submitted.

Test Integration - It is used for development purposes, and verification sessions that are created will not count towards paid usage. Veriff will not provide statuses for sessions created in test integration, you need to set them manually in the Station (see here for instructions). The Webhook decision URL is automatically created.

Vendor -

VENDOR - A business entity SELLING services or goods to Veriff (i.e. not a Client or Customer)

Veriff Station - Veriff's environment, a dashboard where the clients can see their end-users' verification data. Depending on your setup, you may be required to access the environment via station.veriff.com or hub.veriff.com. Please check your sign-up email to make sure that you log in via correct address.

Verification flow - A process where the end-user submits the data required to get verified.

VERIFICATION SESSION - corresponds to unique verification attempt by end_user. One vendor_session can have multiple verification_sessions in case if user has unsuccessful verification attempts which resulted in inconclusive decisions.

Verification session - One verification session from start to end, i.e., from the moment a session is created until the moment a conclusive decision (“approved”, “declined”, “expired”/”abandoned”) is granted. Each verification session receives an unique ID, aka the {sessionID} used in the API request endpoints.

VERIFIED SESSIONS - End user receives a decision from set of {approved, declined, resubmission_requested, review}

Webhook - An event-triggered API, meaning that rather than sending information or performing a function as a response to an application's request, it performs this when a certain trigger event happens.

X-AUTH-CLIENT header - A mandatory header field in API requests, required to identify the request sender. Its value is the API Key (see above).

X-HMAC-SIGNATURE header - A mandatory header field in API requests, required to authenticate the request sender. Its value is a HMAC-SHA256 hex encoded hashed value of the request payload that has been signed using a shared secret key known to both the sending and receiving party.
