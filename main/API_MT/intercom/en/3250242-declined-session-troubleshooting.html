Declined Session Troubleshooting

Here are the reasons why your customer could not be verified

Please note that decline reasons may vary depending on every client's unique Decision Profile


List of reason codes sent via webhooks available here: https://developers.veriff.com/#granular-reason-codes


Reasons for a decline decision:

Suspected document tampering

  • document includes data or features which are not compliant with the appearance of an official specimen

  • document data cross-reference was not successful

Suspicious behaviour

  • portrait photos are shown only from a device screen

  • document photos are shown only from a device screen

  • portrait photos are only presented as a printout or a scan

  • document photos are only presented as a printout or a scan

  • images have been digitally altered

  • there is more than one person visible in the session

Known fraud

  • when the same person, device or document has been present in previous fraudulent sessions

  • slideshow was detected in session video

  • session images are streamed instead of taken in Veriff's flow

The person showing the document does not appear to match the document photo

  • portrait photo does not match document photo

  • more than one unique individual's portrait photos and document photos present in the same session (e.g portrait photo of both Tom and Bob + document photos of both Tom and Bob's documents in the same session)

Velocity Abuse (optional add-on feature and fully-automated decline)

  • velocity/abuse - duplicated user : same user already approved before

  • velocity/abuse - duplicated device : 10 approves from same device have been reached, 11th submitted session from same device declined with this reason.

  • velocity/abuse - duplicated id : same document already approved

If you disagree with Veriff's decision please report it to our Customer Support support@veriff.com
